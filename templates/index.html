How workers work
Each worker is a separate process (or container) running worker.py. Workers poll the job queue and claim jobs automatically. You can run as many as you have resources for.

Worker ID must match the WORKER_ID environment variable set on the container. Adding a worker here also adds it to docker-compose.yml automatically — just run docker compose up -d afterwards to start the new container.

Preferred Worker — pin a job or library to a specific worker (e.g. to use a GPU worker for heavy files).
